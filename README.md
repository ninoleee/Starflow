# Starflow

`Starflow` 是一个面向个人影音库的跨平台播放入口，用来把 `Emby`、`WebDAV`、豆瓣发现、聚合搜索、应用内播放和手动资源匹配收在同一个 App 里。

当前仓库已经是可运行版本，不是纯骨架。

## 开发网络

- Android `Gradle` 已优先使用阿里云 Maven 镜像
- 需要代理或 Flutter 镜像时，参考 [docs/development-network.md](/D:/codes/Starflow/docs/development-network.md)

## 当前能力

### 媒体源

- 支持 `Emby`
  - 登录
  - 读取分区
  - 深层目录浏览
  - `PlaybackInfo` 解析
  - 电影 / 剧集 / 季 / 集详情与播放
- 支持 `WebDAV`
  - 选择路径
  - 读取分区
  - 递归扫描目录
  - 识别目录下常规视频文件
  - 识别 `.strm` 并解析里面的直链
  - 识别同目录 `NFO / poster / folder / cover` 等 sidecar 元数据
  - 识别 `fanart / banner / clearlogo / extrafanart`
  - 支持多层 sidecar 合并
    - 当前文件同目录的 `movie.nfo / 同名 .nfo`
    - 当前季目录的 `season.nfo / index.nfo`
    - 上级剧目录的 `tvshow.nfo / index.nfo`
  - 支持读取 `NFO streamdetails`
    - 容器
    - 视频编码
    - 音频编码
    - 分辨率
    - 码率
  - 支持从文件名 / 目录名里提取 `IMDb ID`
  - 支持从文件名做轻量媒体信息兜底
    - 分辨率
    - 编码
    - 容器
  - 支持两个互不影响的开关
    - `目录结构推断`
    - `本地刮削 / NFO`
  - `目录结构推断` 开启时：
    - 关闭时保持当前文件级扫描
    - 根目录直放且带明显集号的多文件，会当作一个剧的隐式单季
    - 根目录下带季信息的一级子目录会当季
    - 根目录散落文件会当 `特别篇`
    - 没有明显季 / 集信息的泛化目录，不会硬推成剧集结构
  - `本地刮削 / NFO` 开启时：
    - 读取本地 `NFO / poster / fanart / clearlogo` 等 sidecar
    - `NFO` 优先于目录结构推断
  - 扫描后会把多集文件聚合成基础的 `剧 -> 季 -> 集` 结构
  - 扫描完成后建立本地元数据索引
- `Emby` 与 `WebDAV` 现在统一用“已选分区”工作
  - 媒体库只显示已选分区
  - 首页可添加来源只显示已选分区
  - 搜索本地媒体源时只搜索已选分区
  - 手动匹配资源时也只在已选范围里找
- `WebDAV` 首页和媒体库现在优先读取本地索引，不再等进入详情页才补元数据

### 首页

- 首页是模块化的，不是固定布局
- 支持在首页底部进入 `编辑首页`
- 可以添加、删除、启用、拖动排序模块
- 支持添加：
  - 最近新增
  - 任意媒体源的任意已选分区
  - 豆瓣 `我想看 / 我在看 / 我看过 / 随机想看`
  - 豆瓣个性化推荐
  - 豆瓣片单
- 豆瓣模块标题可直接点进完整列表
- `Hero` 支持 `正常 / 无边` 两种样式

### 豆瓣

- 支持：
  - `我想看`
  - `我在看`
  - `我看过`
  - `随机想看`
  - 个性化推荐
  - 自定义片单
- 豆瓣“查看全部”页已支持真实分页加载
- 默认片单预设已内置，可继续自定义输入 URL
- 豆瓣图片会走兼容请求头和本地图片缓存，不再只靠系统直连

### 详情页

- 支持展示：
  - 海报
  - 背景图
  - Logo
  - 年份
  - 评分标签
  - 导演
  - 演员
  - 简介
  - 资源信息
- 剧集详情支持：
  - 选季
  - 横向浏览剧集
  - 直接点集播放
- `WebDAV` 资源默认直接展示扫描期索引结果，不再在详情页自动触发在线刮削
- `WebDAV` 如果开启“目录结构推断”，详情页会直接使用扫描期推断出的季集信息；如果同时开启 `本地刮削 / NFO`，则仍以本地 sidecar 为准
- `WebDAV` 现在不只是文件列表；多集文件会先在索引层聚合成剧集父节点，再进入详情页选季 / 选集
- `WebDAV` 详情页新增了 `建立/管理索引`
  - 会打开全屏索引管理页
  - 可以查看当前本地索引记录
  - 可以手动修改搜索词 / 年份 / 是否按剧集匹配
  - 可以手动搜索 `WMDB / TMDB / IMDb`
  - 选中的结果会直接写回本地索引和详情缓存
- 手动匹配 `Emby` 资源时，会优先按内容类型去更相关的分区找
- 如果匹配到了 `IMDb ID`，会优先按外部 ID 精确命中
- 匹配成功后会显示是按什么匹配到的

### 播放

- 播放器使用 `media_kit`
- 打开播放时会显示轻量等待态
  - 网速
  - 格式
- 播放失败会自动重试最多 `3` 次
- 设置里可以配置“最大打开超时时间”
- 超时后会停止继续尝试

### 搜索

- 支持同时搜索：
  - 本地媒体源
  - `PanSou`
  - `CloudSaver`
- 支持多选搜索来源组合
- 多来源并行搜索，谁先回来先显示谁
- 本地媒体源按来源聚合显示，不再按分区拆多个 tab
- 搜索结果支持：
  - 相同链接去重
  - 按网盘类型筛选
  - 过滤词
  - 强匹配
  - 标题长度上限
- 网盘类型筛选是按返回地址过滤，不靠文字标签兜底
- 在线结果点击即可直接打开资源链接

### 网络存储

- 设置里有单独的 `网络存储` 二级页
- 目前支持：
  - 配置夸克 Cookie
  - 测试夸克连接
  - 选择默认保存文件夹
  - 配置 `SmartStrm Webhook`
  - 测试 `STRM` 任务
  - 配置任务完成后要刷新哪些媒体源
  - 配置刷新延迟秒数
- 搜索结果里的夸克资源支持小按钮直接保存
- 保存成功后可自动触发 `SmartStrm` 任务，并按配置刷新媒体源

### 元数据与评分

- `WebDAV` 媒体源现在会在扫描完成后做后台元数据索引，不把在线刮削挂在详情页
- 资源识别会优先从这些信息里提取干净标题：
  - 文件名
  - 父目录 / 上级目录
  - `SxxExx`
  - 年份
  - 分辨率与发布标签
- `WebDAV` 如果开启“目录结构推断”，会先按目录层级给出基础的内容类型 / 季 / 集判断
- `WebDAV` 的 `目录结构推断` 与 `本地刮削 / NFO` 是两个独立开关
- 后续如果接入正式刮削，刮削结果会覆盖这层结构推断；本地 `NFO` 仍然优先
- `WebDAV` 现在还会优先吃文件名 / 目录名里已有的 `IMDb ID`
- 匹配链路是：
  - 本地 sidecar
  - 文件名 / 目录名里的外部 ID
  - `WMDB`
  - `TMDB`
  - `IMDb`
- `WMDB`
  - 优先用于中文标题 / 简介 / 豆瓣评分
  - 补豆瓣评分
  - 补 `IMDb / 豆瓣 ID`
- `TMDB`
  - 主要用于补全缺失字段
  - 补海报
  - 补简介
  - 补导演 / 演员 / 类型 / 时长
- `IMDb`
  - 只在需要时补 `IMDb` 评分
  - 补 `IMDb` 评分
- sidecar 已有的标题 / 简介 / 海报会优先保留，后续在线来源只补缺失字段
- `WebDAV` 本地 sidecar 现在会优先把这些 artwork 和媒体信息前移到详情页
  - `poster / fanart / banner / clearlogo / extrafanart`
  - `container / codec / resolution / bitrate`
- 如果自动匹配不理想，可以在 `WebDAV` 详情页点 `建立/管理索引`
  - 手动搜索 `WMDB / TMDB / IMDb`
  - 直接选定想要写回的匹配结果
  - 写回后首页 / 媒体库 / 详情页会一起刷新为新的索引结果

## 本地缓存与索引

`WebDAV` 扫描后的元数据索引会保存到本地数据库，记录：

- 来源与路径
- `modifiedAt / size` 指纹
- 识别出的标题 / 年份 / 季集信息
- 识别时使用的搜索词与内容类型偏好
- `sidecar / WMDB / TMDB / IMDb` 命中结果
- `IMDb / 豆瓣` 等外部 ID
- 海报地址 / 简介 / 演员导演 / 评分标签
- `fanart / banner / clearlogo / extrafanart`
- `container / codec / resolution / bitrate`
- 聚合后的剧集父节点与季集关系
- `indexedAt / scrapedAt`
- 最终展示用的 `MediaItem`

详情页也会把这些内容持久化到本地：

- 资源关联结果
- 手动匹配到的资源
- 自动补全的元数据
- 后续刮削得到的信息
- 图片

下次进入相同详情时，会先直接读取缓存，再按缺失字段继续补，不会每次都重新从头查。

`WebDAV` 元数据索引会在这些时机做增量更新：

- 手动刷新媒体源
- 应用内触发媒体源刷新
- 搜索保存成功后触发 `SmartStrm`，并按设置刷新媒体源

如果文件的路径、修改时间和大小都没变，就会直接复用已有索引，不重复请求在线元数据。

媒体库页现在还提供了 `增量更新 WebDAV` 和 `重建 WebDAV 索引` 按钮：

- `增量更新 WebDAV` 会按目录和文件变更做增量刷新
- `重建 WebDAV 索引` 会对当前启用的 `WebDAV` 来源做一次强制全量重扫
- 重建时会跳过已有指纹复用
- 重建时会清掉本轮使用到的在线匹配会话缓存后重新建立索引

除了批量索引，单条 `WebDAV` 资源也支持从详情页手动改索引：

- 手动匹配结果会直接写回本地索引记录
- 同时更新详情缓存
- 首页 / 媒体库下次读取时会直接看到新的索引结果

设置里新增了 `本地存储` 二级页，可以看到当前缓存分类，并支持按类删除：

- `资源关联与刮削信息`
- `图片`

当前设置页里还没有单独暴露 `WebDAV` 元数据索引库的查看和清理入口。

## 快速开始

安装依赖：

```bash
flutter pub get
```

查看设备：

```bash
flutter devices
```

本地运行：

```bash
flutter run -d macos
flutter run -d ios
flutter run -d android
```

指定设备：

```bash
flutter run -d <device-id>
```

## Android

调试安装到安卓手机：

```bash
flutter run -d <android-device-id>
```

打包：

```bash
flutter build apk
flutter build apk --release
```

安装已编译的包：

```bash
flutter install -d <android-device-id>
```

## iOS

`iOS 14+` 上，Flutter 的 `debug` 包不能像普通应用一样从桌面图标直接启动。

建议真机体验时直接用：

```bash
flutter run -d <your-device-id> --profile
```

如果用 Xcode，请打开：

```text
ios/Runner.xcworkspace
```

不要直接打开 `Runner.xcodeproj`。

## 当前实现说明

- `WebDAV` 是当前对 NAS 的统一接入方式
- `WebDAV` 资源的元数据补全已经前移到扫描阶段
- `WebDAV` 扫描现在分两阶段：先快速出目录和文件，再后台补 `NFO / sidecar / 在线元数据`
- `WebDAV` 首页和媒体库读的是本地索引，不是详情页懒加载结果
- `WebDAV` 目录结构推断只是扫描期兜底，后续刮削和 `NFO` 不会和它冲突
- `WebDAV` 目录结构推断和 `本地刮削 / NFO` 是两个分开的开关
- `WebDAV` 目录结构推断现在会优先利用明显的 `Sxx / Exx / Season / 第X季 / 1.日本` 这类命名信息
- `WebDAV` 根目录散落文件会归到 `特别篇`，一级季目录里的文件会归到对应季
- `WebDAV` 多层目录下的 `season.nfo / tvshow.nfo / 同名 nfo` 现在会按层级合并
- `WebDAV` 会优先利用文件名 / 目录名里已有的 `IMDb ID`
- `WebDAV` 会优先吃本地 `fanart / banner / clearlogo / extrafanart`
- `WebDAV` 会优先读取 `NFO streamdetails`，并在没有 sidecar 时用文件名做轻量媒体信息兜底
- `WebDAV` 多集文件现在会先聚合成剧集父节点，再进入详情页看季和集
- `WebDAV` 详情页的元数据修正入口已经改成 `建立/管理索引`
- `WebDAV` 手动修正元数据时，写回的是本地索引，不只是当前详情页临时结果
- `WebDAV` 外部 `.strm` 直链不会继续携带源站 `Basic Auth` 头
- `豆瓣评分` 如果源返回 `0` 或没有值，会按 `豆瓣 0` 显示
- 图片缓存和详情缓存已经接上，清理缓存后会重新拉取
